Production TODO — Puhti Admin & API
=====================================
Last updated: 2026-07-09


🔴 CRITICAL — ALL FIXED 2026-07-09
-----------------------------------------

1. [DONE] Remove debug-db endpoint
   File: admin.py — @router.get('/debug-db')
   The endpoint is auth-protected but still exposes raw DB internals.
   Remove it once the breakdown-by-user bug is confirmed resolved.

2. [DONE] SESSION_SECRET has an insecure default
   File: admin.py line 17 — SESSION_SECRET = os.environ.get('ADMIN_SESSION_SECRET', 'change-me-in-override')
   If deployed without ADMIN_SESSION_SECRET set in override.conf, all admin
   sessions are forgeable. Should crash on startup if the env var is missing.

3. [DONE] GITHUB_TOKEN placeholder in service file
   File: deploy/puhti-run.service — GITHUB_TOKEN=your_token_here
   Real token lives in override.conf but if someone redeploys from the service
   file directly it breaks silently. Remove the placeholder line entirely.

4. [DONE] No rate limiting on submit endpoints
   Files: api_run_endpoint.py — /run-code, /run-notebook
   The 3-job-per-user cap exists but there is no IP or global rate limit.
   A user who keeps polling or submitting can hammer SSH to Puhti.

5. [DONE] /my-jobs and /my-jobs-status have no auth
   File: api_run_endpoint.py — /my-jobs/{username}, /my-jobs-status/{username}
   Anyone who knows a username can poll their jobs or trigger status updates.
   Should require the JupyterHub token header (x-jupyterhub-token).

6. [DONE] 48 rows with NULL username in DB
   File: runs.db — runs table
   Jobs submitted before username was stored show as (anon) in all reports.
   One-time migration: UPDATE runs SET username='unknown' WHERE username IS NULL OR username='';


🟡 SHOULD DO — ALL FIXED 2026-07-09
-------------------------------------

7. SQLite under 4 uvicorn workers
   File: deploy/puhti-run.service — --workers 4
   WAL mode handles concurrent reads but heavy write contention from many
   simultaneous submissions can cause "database is locked" errors.
   Monitor; migrate to PostgreSQL if it becomes an issue.

8. [DONE] SSH connection pooling
   Files: admin.py _ssh(), api_run_endpoint.py _ssh()
   Added ControlMaster=auto + ControlPersist=300 + ControlPath to both
   _ssh() functions. Multiplexes SSH connections; 5-min keepalive.

9. [DONE] Cleanup cron does not prune DB rows
   File: cleanup.sh
   Added DELETE FROM runs WHERE created < date('now', '-90 days') + VACUUM
   at the end of cleanup.sh. Runs daily via cron.

10. [DONE] Job time limit hardcoded in generic_run.sh
    File: generic_run.sh — removed #SBATCH --time=02:00:00
    time_hours param added to /run-code and /run-notebook endpoints.
    Clamped server-side to partition max (longrun=336h, others=72h).

11. [DONE] No persistent log file
    Files: admin.py, api_run_endpoint.py
    RotatingFileHandler added to both — /var/log/puhti-run/api.log,
    10 MB × 5 rotations. Path configurable via API_LOG_FILE env var.

12. [DONE] Admin allowlist hardcoded in source
    File: admin.py — ALLOWED_USERS now reads ADMIN_ALLOWED_USERS env var.
    deploy/puhti-run.service sets ADMIN_ALLOWED_USERS=muhamhamza123,LizCarter492.
    Add new admins via override.conf without a code push.
